## What Each Script Does

### 1. deploy-aws.sh

**Local machine → Launch EC2 instance**

```bash
./scripts/deploy-aws.sh [instance-type] [region]
```

**Steps:**
1. Finds latest Ubuntu 22.04 LTS AMI in us-east-1
2. Creates (or reuses) security group `concentrate-quiz-sg`
3. Adds inbound rules:
   - Port 80 (HTTP)
   - Port 443 (HTTPS)
   - Port 3000 (Next.js frontend)
   - Port 3001 (Fastify API)
4. Launches t3.small EC2 instance with IAM role `sandbox-ssm-instance`
5. Waits for instance to reach "running" state
6. Waits for SSM Session Manager connectivity
7. Outputs Instance ID and connection command

**Output:**
```
✅ EC2 Instance Deployed Successfully!

Instance ID:        i-1234567890abcdef0
Instance Type:      t3.small
Region:             us-east-1
Security Group:     sg-1234567890abcdef0

Next steps:
1. Connect to the instance:
   aws ssm start-session --target i-1234567890abcdef0 --region us-east-1
```

### 2. setup-instance.sh

**EC2 instance → Complete Docker Compose setup**

```bash
./scripts/setup-instance.sh
```

**Steps:**
1. Updates system packages (`apt-get update/upgrade`)
2. Installs Docker and Docker Compose from official repos
3. Installs Node.js 20 (from NodeSource repo)
4. Installs Git
5. Creates app directory (~/concentrate-quiz)
6. Generates .env with random secrets:
   - `JWT_SECRET`: Random 32-byte base64
   - `DB_PASSWORD`: Random 16-byte base64
   - `DB_HOST`: `postgres` (Docker service name)
   - `REDIS_URL`: `redis://redis:6379`
7. Creates `docker-compose.production.yml`:
   - Postgres 17 service
   - Redis 7 service
   - App service (built from Dockerfile.production)
8. Creates `Dockerfile.production` (multi-stage build)
9. Starts all containers with `docker compose up -d`
10. Waits for services to be healthy
11. Displays status summary

**Output:**
```
=== Service Status ===
concentrate-quiz-app       Up 5 seconds (healthy)
concentrate-quiz-db        Up 10 seconds (healthy)
concentrate-quiz-redis     Up 10 seconds (healthy)

=== Health Checks ===
✅ PostgreSQL: OK
✅ Redis: OK
✅ App (Fastify): OK

=== Deployment Complete! ===
```

### 3. teardown-aws.sh

**Local machine → Terminate EC2 instance**

```bash
./scripts/teardown-aws.sh <instance-id> [region] [--delete-sg]
```

**Steps:**
1. Validates instance exists
2. Prompts for confirmation (must type "yes")
3. Terminates EC2 instance
4. Waits for termination to complete
5. Optionally deletes security group (if `--delete-sg` specified)
6. Displays cleanup summary

**Example:**
```
$ ./scripts/teardown-aws.sh i-1234567890abcdef0 us-east-1 --delete-sg

Are you sure you want to terminate this instance? (yes/no): yes
Terminating instance: i-1234567890abcdef0
Waiting for instance to terminate...
State: shutting-down (0/60)
State: terminated (5/60)
Instance terminated!
Deleting security group: sg-1234567890abcdef0

=== Teardown Complete! ===
```

### 4. validate-aws-setup.sh

**Local machine → Check prerequisites**

```bash

```

**Checks:**
1. AWS CLI installed and working
2. AWS credentials configured
3. Default region is us-east-1 (or defaults to it)
4. IAM role `sandbox-ssm-instance` exists
5. Bash shell available
6. Git installed (recommended)
7. jq installed (optional)
8. Deployment scripts present
9. Documentation files present
10. Dockerfile.production present

## Services

### Postgres 17
- Database for all app data (8 tables, migrations)
- Image: postgres:17-alpine (minimal)
- Port: 5432 (Docker internal, not exposed outside container)
- Volume: postgres_data (persistent)
- Healthcheck: `pg_isready -U postgres`
- Environment: DB_USER, DB_PASSWORD, DB_NAME from .env

### Redis 7
- Cache for stats API (60s TTL)
- Image: redis:7-alpine (minimal)
- Port: 6379 (Docker internal)
- Volume: redis_data (persistent)
- Healthcheck: `redis-cli ping`
- Used for: Stats API caching, optional session store

### App (Next.js + Fastify)
- Frontend: Next.js on port 3000
- Backend API: Fastify on port 3001
- Built from: Dockerfile.production (multi-stage)
- Dependencies: postgres (healthy), redis (healthy)
- Environment: All vars from .env
- Healthcheck: `curl http://localhost:3001/health`

## Docker Compose Services

All services are defined in `docker-compose.production.yml` (auto-generated):

```yaml
services:
  postgres:
    image: postgres:17-alpine
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: <generated>
      POSTGRES_DB: concentrate-quiz
    ports:
      - '127.0.0.1:5432:5432'
    healthcheck:
      test: ['CMD-SHELL', 'pg_isready -U postgres']
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    ports:
      - '127.0.0.1:6379:6379'
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 5s
      retries: 5

  app:
    build:
      context: .
      dockerfile: Dockerfile.production
    ports:
      - '127.0.0.1:3000:3000'
      - '127.0.0.1:3001:3001'
    environment:
      NODE_ENV: production
      JWT_SECRET: <generated>
      DB_HOST: postgres
      DB_PASSWORD: <generated>
      REDIS_URL: redis://redis:6379
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_healthy
    healthcheck:
      test: ['CMD', 'curl', '-f', 'http://localhost:3001/health']
      interval: 10s
      timeout: 5s
      retries: 5
```

## Dockerfile.production

Multi-stage production build:

```dockerfile
# Stage 1: Builder
FROM node:20-alpine AS builder
WORKDIR /build
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run type-check
RUN npm run build

# Stage 2: Runtime
FROM node:20-alpine
WORKDIR /app
RUN apk add --no-cache dumb-init curl
COPY --from=builder /build/node_modules ./node_modules
COPY --from=builder /build/.next ./.next
COPY --from=builder /build/src ./src
COPY --from=builder /build/public ./public
COPY --from=builder /build/package*.json ./
ENV NODE_ENV=production
EXPOSE 3000 3001
HEALTHCHECK --interval=10s --timeout=5s --retries=5 \
  CMD curl -f http://localhost:3001/health || exit 1
ENTRYPOINT ["dumb-init", "--"]
CMD ["npm", "run", "dev"]
```

**Result:** ~400 MB production image (optimized)

Generated by setup script with random values:

```bash
# Production Environment
NODE_ENV=production

# JWT Configuration - GENERATED (32-byte base64)
JWT_SECRET=<random-generated>

# PostgreSQL Database - GENERATED
DB_HOST=postgres            # Docker service name
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=<random-generated>
DB_NAME=concentrate-quiz

# Redis Cache - FIXED
REDIS_URL=redis://redis:6379

# Google OAuth (optional)
GOOGLE_CLIENT_ID=           # Optional, empty by default
GOOGLE_CLIENT_SECRET=       # Optional, empty by default
GOOGLE_REDIRECT_URI=http://localhost:3001/auth/google/callback
```

**For production, customize:**
- `JWT_SECRET`: Strong random value (32+ chars), store in AWS Secrets Manager
- `DB_PASSWORD`: Strong random value (20+ chars)
- `GOOGLE_*`: Only if using OAuth
- `DB_NAME`: Optional, defaults to `concentrate-quiz`

## Common Commands

### On Local Machine

```bash
# Validate prerequisites
./scripts/validate-aws-setup.sh

# Launch instance
./scripts/deploy-aws.sh

# Launch with custom size
./scripts/deploy-aws.sh t3.micro us-east-1

# Connect to instance
aws ssm start-session --target i-xxxxx --region us-east-1

# View instance details
aws ec2 describe-instances --instance-ids i-xxxxx

# Terminate instance
./scripts/teardown-aws.sh i-xxxxx us-east-1 --delete-sg
```

### On EC2 Instance

```bash
# View app logs (real-time)
docker compose -f docker-compose.production.yml logs -f

# View specific service logs
docker compose -f docker-compose.production.yml logs app
docker compose -f docker-compose.production.yml logs postgres

# Run migrations
docker exec concentrate-quiz-app npm run migrate

# Test health endpoint
curl http://localhost:3001/health

# View running containers
docker ps

# Access Postgres
docker exec -it concentrate-quiz-db psql -U postgres -d concentrate-quiz

# Stop/restart services
docker compose -f docker-compose.production.yml stop
docker compose -f docker-compose.production.yml start

# View resource usage
docker stats
```

## Security

### Current (Development)
- Security group open to world (0.0.0.0/0)
- Secrets randomly generated (not stored securely)
- No HTTPS (HTTP only)
- No domain (IP-based access)

### For Production
- [ ] Restrict security group to your IP/VPN (0.0.0.0/0 → YOUR_IP/32)
- [ ] Use strong JWT_SECRET (stored in AWS Secrets Manager)
- [ ] Use strong DB_PASSWORD (20+ random chars)
- [ ] Enable HTTPS with Certbot + Let's Encrypt
- [ ] Use custom domain (Route53 DNS)
- [ ] Enable CloudWatch monitoring and logs
- [ ] Regular Docker image updates
- [ ] Implement rate limiting
- [ ] Add database backups (daily snapshots)
- [ ] Consider RDS for Postgres (optional)
